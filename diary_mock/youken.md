## 1. 背景・設計思想

### 1.1 課題

従来の日誌UIは, 生徒が自分の視点で自分を振り返るステップでは「今日の手ごたえ（スライダー）」「今日の気分（アイコン）」「今日の興味（3段階）」で判断することが直接的過ぎて、生徒の自己理解を促す本来の目的を果たしていなかった。

### 1.2 再設計の方針

| 軸 | 方針 |
|----|------|
| 生徒側 | **「自己判断」より「感覚の記録」** — 今の感情と探究心（モチベーション）を、メタファーされた直観的なUIで選択していく |
| 測定妥当性 | Hidi & Renninger（2006, 2022）の行動指標に準拠。スペクトラム（0–100）で連続値として記録し、段階ラベルは目盛りとして保持する |
| 先生側 | 生徒一人ひとりの「感情 × 温度 × AI日誌」を1画面で俯瞰できる構造 |

### 1.3 スコープ外

- 探究チャット本体のUI
- 保護者向け機能
- 管理者（学校）向け機能

## 2. 機能①　生徒：感情 & 温度入力（2ステップ）

### 2.1 概要

日誌生成の2ページ目に表示されるページ。先生への送信はステップ2の最後に行う。

```
チャット終了
    ↓
①AIの視点で振り返る
・今日の観察メモ（確認＆編集）←実装済み
・今日の問い←実装済み
②自分の視点で振り返る
・感情を選ぶ（どんな気持ちで探究してた？）←要修正実装
・温度を選ぶ（探究心はどのくらい燃えてた？）←要修正＆実装
    ↓
送信 → 先生ダッシュボードへ反映
```

---

### 2.2 Step 1 — 感情選択

#### 画面構成

| 要素 | 詳細 |
|------|------|
| ヘッダー | 「今の感情は？」（serif体、中央揃え） |
| 感情グリッド | 8択、2列4行。各セルに感情ラベル＆有機的なフォルムのアイコン＆カラー |
| 選択状態 | 数選択 **可** |

#### 感情選択肢
"emotion_icon_exploration.html" のプロトタイプを参考

#### インタラクション仕様

- タップ → 即時選択（アニメーション: scale 1.0 → 1.05 → 1.0、100ms）
- 再タップ → 選択解除
---

### 2.3 Step 2 — 探究心のモチベーション（炎スペクトラム）

#### 画面構成

| 要素 | 詳細 |
|------|------|
| ヘッダー | 「今の探究心はどのくらい燃えている？」 |
| 炎アニメーション | Rive `.riv` ファイルを使用。State Machine Number Input `heat`（0–100）で制御 |
| モチベーションテキスト | スペクトラム値に連動して変化。serif体、中央揃え |
| スライダートラック | 横軸ドラッグ + 炎エリアの縦ドラッグの2操作に対応 |
| 目盛りラベル | 消えそう / 弱火 / 中火 / 強火（25刻み、段階の参照点として表示） |
| スコア表示 | 0–100の数値をリアルタイム表示 |

#### モチベーションテキスト仕様

| 値の範囲 | テキスト |
|---------|---------|
| 0 – 7 | 面倒くさい。やる気ない|
| 8 – 19 | うーん、ちょっと立ち止まってきたかも。 |
| 20 – 34 | なんか気になってきた。 |
| 35 – 49 | 楽しくなってきた！ |
| 50 – 64 | まだまだやりたい！|
| 65 – 79 | 次の探究の授業が待ち遠しい！ |
| 80 – 100 | 放課後もやる！ |

#### Rive 連携仕様

| 項目 | 値 |
|------|----|
| ファイル | `flame1.riv` |
| State Machine 名 | `FlameControl`（要確認） |
| Number Input 名 | `heat` |
| 入力範囲 | 0.0 – 100.0 |
| フォールバック | `.riv` 未ロード時は絵文字プレースホルダーで動作継続 |

#### インタラクション仕様

- スライダー横ドラッグ → `heat` 値をリアルタイム更新（`pointermove` イベント）
- 炎エリア縦ドラッグ → 上方向で値増加（160px = 100ポイント）
- 値確定 → `heatInput.value = v` で Rive に反映

---

### 2.4 送信データ構造

```typescript
type DiaryEntry = {
  student_id:   string;
  session_id:   string;
  submitted_at: string;           // ISO 8601
  emotion:      EmotionId;        // 'joy' | 'excited' | ...
  heat_score:   number;           // 0–100（整数）
  ai_diary:     AiDiary;          // AIが生成（チャットログから）
};

type AiDiary = {
  observation:  string;           // AI観察文（生徒が編集可）
  quote:        string;           // チャット内の印象的な発言
  next_question: string;          // 次回探究への問い
};
```

---

### 2.5 画面遷移

```
[Step 1: 感情選択]
  感情タップ → 選択状態変化

[Step 2: 炎スペクトラム]
   スライダーを操作 → 焔の状態とラベル変化変化
  「先生に届ける」タップ → 送信 API コール → 完了画面
```

---

### 2.6 アクセシビリティ

- 感情グリッド: `role="radiogroup"` / 各セル `role="radio"`
- 炎スライダー: `<input type="range">` をベースに実装（Riveは視覚補助のみ）
- キーボード操作: Tab / Arrow Key でスライダー値変更可能

## 5. 非機能要件

### 5.1 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| Riveアニメーション | 60fps（モバイル含む） |
| スライダー入力遅延 | `pointermove` → Rive反映 < 16ms |
| ダッシュボード初期表示 | < 1.5s（30人クラス） |

### 5.2 対応環境

| 環境 | 対象 |
|------|------|
| 生徒側 | iOS Safari 16+, Chrome Android 110+ |
| 先生側 | Chrome 110+, Safari 16+（デスクトップ優先） |
| Rive | `@rive-app/canvas@2.21.6` 以上 |

### 5.3 フォールバック

- Riveアセット未ロード → 絵文字プレースホルダーで全機能動作継続
- オフライン → 送信ボタン非活性 + 「接続を確認してください」トースト
