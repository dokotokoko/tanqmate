# 探Q LAB 実装引継資料

## 概要

「探Qメイトサービス空間のLAB化」として、**あつまれ動物の森の俯瞰カメラ視点を忠実に再現した3Dアイソメトリック研究室ページ**を実装しました。ユーザーが部屋の中を歩き回り、壁に掛けられたアイテムをクリックして探究活動のデータを確認できるゲーム空間です。

---

## v2 アーキテクチャ（アイソメトリック3D部屋）

### 部屋のレンダリング方式

**SVG viewBox (1000×700) ベースの2.5Dルーム**で、以下のポリゴンにより3D空間を構成します。

```
         天井 (#7A6E60)
(0,0)──────────────────────────(1000,0)
  \    (150,20)────────(850,20)    /
   \   │     背面壁 (レンガ)    │   /
    \  │  [時計] [モニター] [ボード] │  /
     \ │                        │ /
(0,90) (150,300)────────(850,300) (1000,90)
  │  左壁  │                  │  右壁  │
(0,390)    │     巾木ライン    │    (1000,390)
  │        │                  │        │
  │        床（木目+カーペット）          │
  │   [本棚] [小テーブル] [アバター]     │
  │                     [デスク] [椅子]  │
(0,700)────────────────────────(1000,700)
```

**座標系:**
- 背面壁: `(150,20) → (850,20) → (850,300) → (150,300)`
- 左壁: `(0,90) → (150,20) → (150,300) → (0,390)`
- 右壁: `(850,20) → (1000,90) → (1000,390) → (850,300)`
- 床: `(0,390) → (150,300) → (850,300) → (1000,390) → (1000,700) → (0,700)`

### アバター移動システム

| 操作 | 挙動 |
|---|---|
| 床クリック | クリック地点にアバターが滑らかに移動（cubic-bezier遷移 0.6秒） |
| 矢印キー↑↓←→ | 15px刻みで移動、床ポリゴン境界内に制限 |
| 歩行アニメーション | 移動中は足が上下する SVG animate を表示 |
| パース縮尺 | `pScale(y)` 関数: 奥(y=300)→0.45倍、手前(y=700)→1.0倍 に自動スケール |

**床境界判定:** Ray Casting アルゴリズムで点がポリゴン内にあるか判定。

### 壁面インタラクション（概要 → 詳細）

| 壁面アイテム | 表示される概要情報 | クリックで開くモーダル |
|---|---|---|
| 大型モニター（背面壁中央） | プロジェクト名、進捗 XX%、8ステップのドット | `ProgressMapDisplay` |
| コルクボード（背面壁右） | MBTI風タイプコード (例: EPIA)、タイプ名 | `PersonalityPanel` |
| 額縁ポスター（左壁） | 質問回数、連続活動日数 | `EffortTimelinePanel` |

壁面アイテムには**パルスグロー効果** (`<animate>`) が常時表示され、クリック可能であることを視覚的に示します。

## バックエンド実装（前回から変更なし）

### APIエンドポイント

| メソッド | パス | 説明 |
|---|---|---|
| GET | `/lab/stats` | ユーザーの全体統計 |
| GET | `/lab/progress` | 全プロジェクトの進捗状況（8ステップ） |
| GET | `/lab/personality` | 学習パーソナリティ分析（MBTI風4次元） |

### 探究レポート進捗の8ステップ

1. テーマ設定 → `theme`フィールド有無
2. 問いの設定 → `question`フィールド有無
3. 仮説の構築 → `hypothesis`またはメモ内キーワード
4. 情報収集 → 調査キーワード + メモ数
5. 分析・考察 → 分析キーワード + メモ数
6. 結論の導出 → 結論キーワード
7. レポート作成 → レポートキーワード
8. 発表準備 → 発表キーワード

### 学習パーソナリティ（MBTI風4次元 × 16タイプ）

| 次元 | Pole A | Pole B |
|---|---|---|
| 探索/専門 | 探索型 (E) | 専門型 (S) |
| 理論/実践 | 理論型 (T) | 実践型 (P) |
| 独立/対話 | 独立型 (I) | 対話型 (C) |
| 創造/分析 | 創造型 (R) | 分析型 (A) |

## 今後の課題・改善案

### 優先度: 高

1. **ステップ完了判定の精度向上**
   - LLMによる自然言語解析で判定精度を向上
   - ユーザーが手動でステップを完了マークする機能

2. **パーソナリティ分析の改善**
   - `chat_logs`に`response_style_used`カラム追加で確実に記録
   - LLMによる質問傾向の詳細分析

3. **参照URL一覧の表示**
   - メモ・チャットからURL抽出、壁面「参考文献ボード」追加

4. **アバターと家具の衝突判定**
   - 現在アバターは家具の上を通過できてしまう
   - 家具の占有矩形を定義し、移動可能領域から除外する

5. **モバイル対応の改善**
   - 現在のSVGルームはモバイルでタップ操作は可能だが画面が小さい
   - ピンチズームまたはスクロール対応の検討
   - モバイル専用の簡略化ビュー（カード型UIへのフォールバック）

### 優先度: 中

6. **部屋のカスタマイズ機能**
   - 壁紙、床、家具を選べるインテリアコーディネート
   - `user_lab_config`テーブル新設

7. **アチーブメントバッジシステム**
   - 壁面「トロフィー棚」、既存クエストシステム連携

8. **アバターのカスタマイズ**
   - 髪型、服装、アクセサリーのパーツ選択

9. **プレイ時間の正確な計測**
   - Page Visibility APIによるフロントエンド計測

10. **家具のインタラクション強化**
    - デスクのノートPCクリック → AIチャットへ遷移
    - 本棚クリック → メモ一覧を表示
    - 植物に水やり → 連続ログインボーナス演出

### 優先度: 低

11. **BGMとサウンドエフェクト**（Web Audio API）
12. **他ユーザーの部屋訪問機能**
13. **季節イベント連動**（桜・紅葉・雪の装飾変化）

## 技術的な注意事項

### フロントエンド

- **SVG viewBox**: `0 0 1000 700` で固定比率。`preserveAspectRatio="xMidYMid meet"` によりレスポンシブ対応
- **座標変換**: `clientToSvg()` 関数でマウスクリック座標をviewBox座標に変換
- **境界判定**: `ptInPoly()` で Ray Casting アルゴリズムを使用
- **パース縮尺**: `pScale(y)` で奥行きに応じたキャラクター・影のスケーリング
- **SVGアニメーション**: `<animate>` タグでパルスグロー、足の歩行を表現
- **CSS transition**: アバター移動は `transition: transform 0.6s cubic-bezier(...)` で滑らかに
- **矢印キー操作**: `window.addEventListener('keydown')` でグローバルにキャプチャ、モーダル開閉中は無効化
- **パネル詳細**: 既存の `ProgressMapDisplay`, `EffortTimelinePanel`, `PersonalityPanel` をMUI Dialogモーダル内で表示

### バックエンド

- `LabService`は`BaseService`を継承
- Supabase直接クエリ（既存パターン準拠）
- JWT Bearer認証（`get_current_user`依存関数）

### DB（追加テーブル不要）

- 既存テーブル (`chat_logs`, `projects`, `memos`, `chat_conversations`) のみで動作

## 動作確認手順

1. Docker Compose でバックエンド・フロントエンドを起動
2. ログイン後、左サイドバーの「探Q LAB」をクリック
3. あつ森風の3D部屋が表示されることを確認
4. **床をクリック** → アバターがその位置に歩くことを確認
5. **矢印キー** → アバターが移動すること、壁や家具の外に出ないことを確認
6. **背面壁のモニターをクリック** → プロジェクト進捗モーダルが開くことを確認
7. **コルクボードをクリック** → パーソナリティ詳細モーダルが開くことを確認
8. **左壁の額縁をクリック** → 努力の軌跡モーダルが開くことを確認
9. 操作ガイド（画面下部）が5秒後に自動的にフェードアウトすることを確認
