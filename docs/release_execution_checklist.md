# 探Qメイト リリース実行チェックリスト

この文書は [release_plan.md](/abs/path/C:/Users/kouta/tanqmates/docs/release_plan.md) を実行に落とすための別紙です。

方針よりも、各フェーズで何を終わらせるか、何をもって完了とするかを明確にすることを目的にしています。

2026-04-13 時点で、現行コードベースを確認し、実装が明確に確認できたタスクだけ ` [x] ` を付けています。

## 使い方

- 各フェーズで、`未着手 / 進行中 / 完了 / 保留` の状態を付ける
- タスクを閉じる前に、必ず完了要件を満たしているか確認する
- Go / No-Go 判定は、Phase 1 と Phase 3 の完了要件を満たしてから行う
- 実装より先に、法務・運用・障害対応の前提を崩さない

### タスクリスト

- [ ] 生成失敗時の再試行を実装する
- [ ] 結果の編集を可能にする
- [ ] 手動入力へのフォールバックを用意する
- [x] 保存成功 / 失敗を明示する

#### 3. 共有サマリー v0

- [ ] 生徒に共有前確認画面を出す
- [ ] 学校側には raw ではなく summary のみを表示する
- [ ] 共有済み / 未共有の状態を本人に分かる形で表示する
- [ ] 先生向け画面に「summary 表示」であることを明記する
- [ ] 共有の取り消し可否と扱いを仕様化する

#### 4. 品質 / 観測 / 運用

- [x] 問い合わせ / 不具合報告導線を置く

### 完了要件

- 新規ユーザーが `登録 → オンボーディング → 初回日誌生成 → 保存 → 共有確認` を通しで完了できる
- 主要画面で empty / loading / error が破綻なく表示される
- 日誌生成失敗時に再試行か手動入力へ必ず退避できる
- 学校側に raw データが表示されない仕様と画面が確認できる
- 問い合わせ導線、障害時の一次対応、手動フォールバックが文書化されている
- 利用規約、プライバシーポリシー、未成年利用の運用方針が最低限整っている
- 指標定義と Go / No-Go 条件が確定している

### Go / No-Go 判定項目

- `登録完了率`
- `オンボーディング完了率`
- `初回日誌生成到達率`
- `初回共有完了率`
- `生成失敗率`
- `7日以内再訪率`
- `不具合報告数`
- `続けたい / なくなると困る` の定性反応

注記:
各指標には、別途 `閾値 / 集計期間 / 分母 / セグメント` を定義すること。

- [ ] Discord / Slack / LINE オープンチャットのどれか1つに絞る

- 少なくとも1つの cohort で、募集から初回利用、感想回収、改善反映までの運用が一巡している
- 運営チャネル、アンケート、共有会のリズムが固定されている

- 毎週の改善ログが継続公開できている

## 学校現場での実験計画

### 目的

導入ではなく、6週間前後の小規模実証として、価値と安全性の両方を検証する。

### タスクリスト

#### 1. 実証設計

- [ ] 1校あたり champion teacher を1人置く
- [ ] 対象を1クラスまたは1学年の小さめ単位に絞る
- [ ] 6週間の実施スケジュールを組む
- [ ] 週1回以上の活用機会を設計する
- [ ] 事前説明、導入研修、週次ふり返り、事後報告の流れを固める

#### 2. 仮説設計

- [ ] 学校A / 学校B で試すテーマを分ける
- [ ] `整理` と `問い返し` が効くかを検証する
- [ ] 生徒確認つき共有サマリーが抵抗感を下げるか検証する
- [ ] summary のみで先生が介入できるか検証する
- [ ] 週次サマリーで伴走の質が上がるか検証する

#### 3. 取得データ

- [ ] 定量指標を定義する
- [ ] 定性インタビューの観点を定義する
- [ ] 共有前編集率やエラー率の取り方を決める
- [ ] 実証後比較のための基準点を決める

#### 4. 運用品

- [ ] 教員向け導入スライドを用意する
- [ ] 生徒向け利用ルールを用意する
- [ ] 保護者向け説明文を用意する
- [ ] 問い合わせ窓口を明記する
- [ ] 週次の実証レポート雛形を作る
- [ ] 実証後のケーススタディ雛形を作る

### 完了要件

- 参加校ごとに担当教員、対象範囲、実施期間、目的が明確になっている
- 仮説、取得データ、振り返り方法が事前に定義されている
- 教員 / 生徒 / 保護者向けの基本説明資料が揃っている
- 週次レポートと事後報告を無理なく継続できる
